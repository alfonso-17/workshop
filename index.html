SOLUCION

1. ¿Qué es un control de versiones?



Sistema que registra cambios en archivos a lo largo del tiempo, permitiendo recuperar versiones específicas y facilitando la colaboración

2. ¿Qué es GIT?



Sistema de control de versiones distribuido que permite a múltiples desarrolladores trabajar en un proyecto simultáneamente sin interferir en el trabajo de los demás.

3. ¿Qué es un repositorio?, repositorio locales y remotos.



Local: Repositorio en tu máquina local.
Remoto: Repositorio en un servidor remoto (ej. GitHub) para compartir y colaborar.

4. ¿Qué es GITHUB?



Plataforma de desarrollo colaborativo que aloja proyectos utilizando Git y proporciona herramientas para colaboración y gestión de versiones.

5. ¿Qué es una rama en GIT y para qué sirve?



Línea de desarrollo independiente que permite trabajar en características o correcciones de errores sin afectar la versión principal del proyecto.

6. ¿Cuáles son los estados en GIT?



Modificado: Archivo cambiado pero no preparado. Preparado: Cambios listos para ser confirmados. Confirmado: Cambios guardados en el historial del repositorio.

7. ¿Qué es la terminal, consola o línea de comandos y para qué sirve?



Interfaz basada en texto para interactuar con el sistema operativo y ejecutar comandos.

8. Escribir el funcionamiento de cada uno de los siguientes comandos:



git init: Inicializa un nuevo repositorio. git clone : Clona un repositorio. git status: Muestra el estado del repositorio. git add : Añade archivos al área de preparación. git commit -m "mensaje": Confirma los cambios. git push : Envía cambios al repositorio remoto. git pull : Recupera cambios del repositorio remoto. git branch : Crea una nueva rama. git checkout : Cambia de rama. git merge : Fusiona ramas.
